HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user
  • Elements nested inside other elements are called child elements
  • The element that holds the children is the parent element
  • An element refers to the entire element from opening to closing tag, whereas a tag refers only to what's inside the angle brackets
  • The h1 to h6 elements are used for representing the level of heading (most prominent to least)
  • ul, ol, li, represent unordered list, ordered list, and list items, respectively
  • Add alt text to your images for those that use screen readers or browsers that don't display the image. Also for search engine optimization (SEO)
  • A margin indicates how much space wanted around elements
  • A padding indicates how much space wanted around content inside element

CSS

  • Inline CSS: used to style directly in HTML file
  • Apply to specific HTML tag using attribute with CSS rule to style particular page element only
  • Useful for quick, permanent changes; less flexible than external and internal style sheets bc of individual editing of each style
  • Internal Style Sheet (ISS): embed entire styleshee directly in HTML file thru style element
  • ISS Applies to specific page, can style multiple elements in the page code; CSS rules used will apply to all page elements
  • External Style Sheet (ESS): most common and preferred. Allows all CSS rules to be kept in separate file, making design changes easier.
  • ESS separate file is linked to HTMLs head element using link element
  • Separation of concerns: each section of code should have its own responsiblity so separate code in different files based on use

GIT

  • git status: checks what branch we are currently in
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin margin: pulls in base branch from GitHub
  • git checkout main: swithces to branch named main
  • git add -A: commits all files in the working branch
  • git commit -m: adds message to committ. Keep short, descriptive
  • git push origin: pushes changes to GitHub repo to stage pull request

JavaScript

  • Allows users to interact with websites (effects, interactivity, data communication, etc)
  • Control Flow: code is read and executed from top to bottom, order matters
  • Conditional Statement: can be used to manipulate the Control Flow, ie, execute this code only if certain condition is met
